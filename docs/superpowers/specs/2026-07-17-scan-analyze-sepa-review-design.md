# ticker-scan → stock-analyze 一致性 review 机制设计

**日期：** 2026-07-17
**触发案例：** PATH 被 `/ticker-scan` 初筛标为"本轮唯一价格结构健康标的"，但完整 SEPA 8 条模板核查后发现 MA150($12.21)、MA200($12.95) 都在现价($12.12)之上，且 MA200 仍在下降——这是 Stage1→2 早期未确认的底部反转，不是 Stage2 健康趋势。

## 根因

`/ticker-scan` Stage3 的 2C 标准#5（价格结构核查）用简化启发式 `ma50_pct > 0 AND ma200_pct > -20`，跟 `/stock-analyze` 用的完整 8 条 SEPA 趋势模板判断逻辑本来就不对齐。`scripts/ticker_scan.py` 也没有计算 MA150、没有 MA200 趋势方向，缺了判断 Stage1 vs Stage2 所需的最小字段。这是方法论定义层面的漂移，不是随机数据误差——同类漂移未来可能出现在其他判断项（backlog 解读、技术壁垒判断等 LLM 定性项）上。

## 方案：规则否决 + 精准抽查（对比方案 A 纯文档最小改动、方案 B 全面独立 agent 复核后选定）

规则否决表覆盖广、零成本，兜住系统性偏差；独立 agent 复核只用在最终 🔥 候选（通常 1-3 只，样本少），把复核成本花在刀刃上。

## 改动明细

### 1. `scripts/ticker_scan.py` — 新增两个字段

在现有 `fetch_ticker()` 中，复用已抓取的 `period="1y"` 历史数据（约 252 个交易日，够算 MA200 趋势所需的 221 天窗口）：

- `ma150_pct` — 现价相对 MA150 的百分比。历史 <150 天 → 字段省略（沿用现有 `ma200_pct` 的省略风格）。
- `ma200_trend` — `"up"` / `"down"` / `"flat"`。比较当前 MA200 与 21 个交易日前的 MA200，变化幅度 >+0.5% → `"up"`，<-0.5% → `"down"`，否则 `"flat"`。历史 <221 天 → 字段省略。

不实现完整 SEPA 第 8 条（相对强度百分位，需要全市场横向排名，扫描阶段成本过高，超出本次范围）。

更新模块 docstring 里的字段列表说明。`print_result()` 文本输出模式同步加两列。

### 2. `.claude/commands/ticker-scan.md` Stage3 2C 标准#5 — 重新定义

旧：`ma50_pct > 0 AND ma200_pct > -20` → ✅/⚠️/❌

新：四条子检查（对应 SEPA 8 条模板中最容易被 Stage1 反转案例打破的部分）：
1. `ma50_pct > 0`（价 > MA50）
2. `ma150_pct > 0`（价 > MA150）
3. `ma200_pct > 0`（价 > MA200）
4. `ma200_trend != "down"`

判定：
- 4 条全满足 → ✅ **Stage2健康趋势**
- 部分满足（比如价已站上 MA50 但 MA150/MA200 还没突破，或刚转平）→ ⚠️ **Stage1→2过渡**
- `ma150_pct < 0` 或 `ma200_trend == "down"` → ❌ **Stage1未确认底部反转**（明确用这个措辞，不再用"价格结构健康"这类容易和 Stage2 混淆的说法）

数据不足（字段缺失）→ ⚠️，注明"MA150/MA200趋势数据不足，需完整 SEPA 核实"。

### 3. Step 6 打标签 — 否决表（新增小节，紧接现有标签规则表之后）

不论 2A/2B 分数多高，2C标准#5 结果一票否决：

| 2C标准#5 结果 | 否决动作 |
|---|---|
| ❌ Stage1未确认底部反转 | 标签封顶 **👀观察**，即使 2B类型/公式/2C其余四条都很强 |
| ⚠️ Stage1→2过渡 且本应判 🔥 | 降为 **⭐翻倍候选** |

报告"综合排名"表的"标签"列，若发生降级，加一句降级原因（如"MA200仍下降，Stage1底部反转未确认"）。

### 4. Step 6.5（新增）— 独立 agent 复核，仅对否决表跑完后仍为 🔥 的候选

对每个最终 🔥 候选（通常 1-3 只），用 Agent 工具起一个独立子 agent（干净上下文，不携带本次 scan 的推理过程），只喂：ticker、现价、MA50/150/200 相对百分比、MA200趋势、EPS trend、revenue_yoy、backlog 摘要（若 Stage2/3 已 WebSearch 到）。要求它独立套用 SEPA 逻辑给出 Stage 判断（1-4）+ 一句话理由。

- 独立判断与 scan 结论一致 → 无需额外标注
- 不一致 → 报告里在该候选行加 "⚠️独立复核不一致，建议 `/stock-analyze` 前先人工确认"；标签本身不强行覆盖（人是最终判断者，只是多一层信号）

### 5. 输出格式微调

"综合排名"表 2C核查 列（5 个 emoji）里，标准#5 位置保留但语义已更新为上述四条聚合结果；"建议"列若触发独立复核不一致警示，追加提示文字。`Stage 1/2 筛除` `新发现` `赛道结论` 等其余章节不变。

## 不做的事（明确排除，避免范围蔓延）

- 不实现完整 SEPA 第 8 条相对强度百分位
- 不对所有 ⭐/👀 候选做独立 agent 复核（成本考量，只覆盖 🔥）
- 不改动 `/stock-analyze` 本身的完整 SEPA 8 条模板判断逻辑（那是权威版本，scan 阶段是向它对齐，不是反过来）
- 不新增独立 review 命令/skill；复核步骤内嵌在 `/ticker-scan` 流程里

## 影响范围

- `scripts/ticker_scan.py`：加两个字段，向后兼容（新字段可能为 `None`/省略，不影响现有消费方）
- `.claude/commands/ticker-scan.md`：Stage3 标准#5 定义、Step6 打标签规则、新增 Step6.5
- 不影响 `/stock-analyze`、`/stock-entry`、`/stock-exit`、`/morning-check` 等下游 skill
