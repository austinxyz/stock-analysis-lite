# stock-entry

基于 SEPA + Moneyball 框架，给出翻倍候选股的具体入场区间、止损位、仓位建议和建仓计划。

## Arguments

Required: `<TICKER>` — 股票代码（如 `ABAT`）

## Steps

### 1. 读取分析报告

检查 `wiki/tickers/[TICKER]/analysis.md` 是否存在，提取：
- 当前 Stage 和趋势模板得分
- 价格 vs 各 MA 数据
- ATR14、近期催化剂

如无此文件，先运行 `/stock-analyze [TICKER]`。

---

### 2. 拉取当前数据（共用数据层）

```bash
python scripts/ticker_scan.py TICKER --mode full --benchmark --json
```

使用字段：`price`、`atr14`、`atr_pct`、`ma50_pct`/`ma150_pct`/`ma200_pct`、`vol_avg20_m`、`vol_ratio`、`trend_template.score`、`earnings_in_days`、benchmark 行 `market_env`。

---

### 3. 入场条件确认（SEPA）

- [ ] Stage 2 上升趋势（MA50 > MA150 > MA200，价格在 MA50 上方）
- [ ] 趋势模板得分 ≥ 3/8
- [ ] 有明确轴心点（平台突破口 / VCP / 杯柄形态）
- [ ] 突破时成交量放大（量比 ≥ 1.5×）
- [ ] 财报距离：`earnings_in_days > 14`（≤14 → ⚠️ SEPA 规则：财报前 2 周不入场，结论标"等财报后"）
- [ ] 市场环境（`market_env`）：`bear` → ❌ 不建新仓（SEPA 第六节主开关）；`chop` → 单笔风险降至 0.5%；`bull` → 1%

**入场区间：** 轴心价 → +5% 为买入区（0-3% 理想）。超过 +5% 不追，等下次形态（SEPA 原典口径）。

---

### 4. 止损计算

翻倍候选使用 ATR 自适应止损（不用固定百分比，避免日波幅大的小盘股被洗出）：

```
止损宽度 = max(2 × ATR14, 入场价 × 10%)
止损位   = 入场价 - 止损宽度
上限：止损宽度不超过入场价的 20%
```

**论文止损（优先级更高）：**
- 核心催化剂消失 → 立即出
- 稀释融资 >20% 且无对应业务价值 → 立即出
- 现金跑道 <3 个月 → 评估是否出

---

### 5. 目标价 + R/R 计算

```
目标1（轻减 25-30%）= 入场价 × 1.30~1.50   (+30-50%)
目标2（翻倍区）    = 入场价 × 1.80~2.50   (+80-150%)
自由股触发         = 入场价 × 2.00         (+100%)

R/R(T1) = (目标1 - 入场价) / (入场价 - 止损价)  → 要求 ≥ 2:1
```

**Moneyball PW EV 触发线（必查）：**
读 `wiki/tickers/[TICKER]/analysis.md` 的「Moneyball 情景 EV」节：
- 现价 ≤ EV × 0.85 → ✅ 有安全边际，可执行入场
- 现价 > EV × 0.85 → ❌ 不追，等回调或等 EV 重估（synthesis.md 优先级规则 #5）
- analysis.md 无 EV 节 → 先重跑 `/stock-analyze [TICKER]`

**自由股规则：** 达到 +100% 时 → 卖出 50% 锁利，剩余 50% 零成本持有，目标 3-5×

---

### 6. 仓位大小（风险预算法）

```
单笔风险 = 总资金 × R%（R 由 market_env 决定：bull=1%，chop=0.5%，bear=不建仓）
建议股数 = (总资金 × R%) ÷ (入场价 - 止损价)
```

示例：总资金 $10,000，入场 $10，止损 $8.50（止损幅度 $1.50）
→ 建议股数 = ($10,000 × 1%) ÷ $1.50 = 67 股

---

### 7. 金字塔式建仓（三步）

Section 6 计算的股数 = 总计划股数，分三批入场：

```
批次1（试仓 40%）
  → 时机：轴心点突破 + 量比 ≥ 1.5×
  → 目的：小仓验证论点，2日内若跌破止损则论点证伪

批次2（确认加仓 40%）
  → 时机：批次1入场价上涨 8-12% 后追入
  → 原则："不怕追高"——上涨是论点确认信号，不是错过信号

批次3（加码 20%）
  → 时机：催化剂事件确认（财报/合同/产品验证）
  → 可选：批次1+2已充分则跳过
```

> **无杠杆铁律：** 三批合计不超过总资金风险上限，绝不借贷。

---

### 8. 输出入场计划

保存到 `data/tickers/[TICKER]/entry-[YYYY-MM-DD].md`：

```markdown
# [TICKER] 入场计划 — YYYY-MM-DD

## 当前状态
- 现价：$X | Stage：X | 趋势模板：X/8
- 入场条件：[✅ 满足 / ⚠️ 部分满足 / ❌ 不满足]
- ATR14：$X（当前价的 X%）
- 市场环境：[bull / chop / bear]（SPY vs MA200 X% | QQQ X%）→ 单笔风险 R%
- 财报距离：X 天 [✅ >14 / ⚠️ ≤14 等财报后]
- PW EV 对照：现价 $X vs EV×0.85 = $X → [✅ 可入 / ❌ 不追]

## 入场方案

| 项目 | 价位 | 说明 |
|------|------|------|
| 入场区间 | $X – $X | 轴心点上方 0-3% |
| 止损位 | $X（-X%）| max(2×ATR=$X, 10%) |
| 目标1（减仓 25-30%）| $X（+X%）| 留大仓跑翻倍 |
| 目标2（翻倍区）| $X（+X%）| |
| 自由股触发 | $X（+100%）| 卖 50%，剩余零成本持有 |
| R/R（T1）| X:1 | |

## 仓位建议
- 总资金：$X | 单笔风险（1%）：$X
- 建议总股数：X 股（$X 市值）
- 占总资金：约 X%

## 建仓计划（金字塔式）

| 批次 | 比例 | 价格触发 | 股数 | 触发条件 |
|------|------|---------|------|---------|
| 批次1（试仓）| 40% | $X | X 股 | 轴心点突破 + 量比 ≥1.5× |
| 批次2（确认）| 40% | $X（+8-12%）| X 股 | 价格确认上涨后追入 |
| 批次3（加码）| 20% | 催化剂触发 | X 股 | 财报/合同/产品验证 |

> **止损位 $X 适用全部批次。**

## 关键催化剂
[触发上涨的核心事件]

## 结论
[现在可入 / 等待形态 / 条件未满足原因]

入场后用 `/stock-exit [TICKER]` 管理持仓。
```
